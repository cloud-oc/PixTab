# Pixiv 图片反向代理使用指南

## 简介

对于中国大陆或某些网络受限地区的用户，直接访问 Pixiv 图片服务器 `i.pximg.net` 可能会失败或速度较慢。本扩展支持反向代理功能，可以通过第三方镜像站点加载图片。

## 使用方法

1. 打开扩展的设置页面
2. 滚动到 **高级设置** 部分
3. 在 **使用反向代理** 输入框中，填入以下任一可用的反向代理域名：
   - `i.pixiv.re` (推荐)
   - `i.pixiv.cat`
   - `i.pixiv.nl`
   - 或其他可用的 Pixiv 镜像域名
4. 留空则不使用反向代理
5. 保存设置后，扩展会自动使用反向代理加载图片

## 工作原理

当设置了反向代理域名后，所有对 `i.pximg.net` 的图片请求会被自动替换为您设置的反向代理域名。例如：

原始 URL:
```
https://i.pximg.net/img-original/img/2024/01/01/00/00/00/123456_p0.jpg
```

替换后的 URL (使用 i.pixiv.re):
```
https://i.pixiv.re/img-original/img/2024/01/01/00/00/00/123456_p0.jpg
```

## 推荐的反向代理服务

### i.pixiv.re
- 速度快，稳定性好
- CDN 覆盖全球
- 默认推荐选项

### i.pixiv.cat
- 老牌镜像站
- 使用 Cloudflare CDN
- 访问稳定

### i.pixiv.nl
- 备用选项
- 部分地区速度较好

## 注意事项

1. **反向代理服务由第三方提供**，请根据实际情况选择可用的服务
2. 如果某个代理域名无法访问，可以尝试更换其他域名
3. 反向代理仅用于加载图片，不影响 Pixiv API 的访问
4. 如果您已经可以正常访问 Pixiv，不需要使用此功能（留空即可）

## 故障排除

### 图片无法加载
- 尝试更换其他反向代理域名
- 检查您的网络连接
- 确认反向代理服务是否正常运行

### 加载速度慢
- 不同的反向代理服务在不同地区的速度可能有差异
- 可以尝试切换到其他代理域名
- 部分反向代理可能有访问频率限制
